About

This starter uses static site generation (SSG): your pages become fast, standalone HTML you can deploy anywhere.

What is SSG?

With SSG, your site is rendered at build time. Visitors get a real HTML page (great for performance and SEO), and you can still add JavaScript later when you need interactivity.

How Webstir helps

  • Keep pages simple: plain HTML and CSS per page.
  • Share a layout: common markup lives in src/frontend/app/app.html.
  • Ship production output with webstir publish.

Dev experience

Run webstir watch while you edit for fast refresh and hot reload, so you can iterate without a full rebuild.

Build output

  • Dev builds go to build/frontend/ (used by webstir watch).
  • Publish output goes to dist/frontend/ (used by webstir publish).

Deploying

dist/frontend/ is a plain static site. If you want clean URLs (like /docs/content-pipeline/), configure your host to serve index.html for directory paths.
